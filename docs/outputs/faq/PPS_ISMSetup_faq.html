PPS – ISM Setup – Frequently Asked Questions

This FAQ supports the M1. ISM Setup – At PPS Service SOP. It explains how to use the ISM Setup hub to configure partners, schedules, work codes, warranty GL, service order templates, warranty codes, rates, and contract templates before they are used in day-to-day service work.

Questions & Answers

1. What is the purpose of the ISM Setup SOP at PPS Service?

The ISM Setup SOP provides a single hub for setting up key ISM master data used by PPS Service. It covers:

  • Partner setup (service partners / vendors).
  • Partner schedule setup.
  • Work codes.
  • Warranty-specific GL accounts.
  • Service order templates (SRO templates).
  • Warranty codes.
  • Rates.
  • Contract templates for rental and service contracts.

Using this SOP helps ensure that all of these elements are set up consistently before they are used in transactional SOPs such as Start-Up, Rental, or PMA.

2. Where does this SOP start and end?

The process starts when a user recognizes that a master data element needs to be set up or changed (for example, a new partner, new work code, or new contract template) and navigates into the ISM Setup SOP.

It ends after the appropriate setup step (S1–S9) is completed and the user either uses S998. Restart Selection to perform another setup task or exits back to their transactional SOPs. In this way, ISM Setup functions as a configuration companion to day-to-day service processes.

3. How does the SOP help with Partner setup (S1) and Partner schedules (S2)?

The SOP includes distinct steps for partners and their schedules:

  • S1. Partner Set Up – used when PPS needs to add or maintain a partner, such as an external service provider or subcontractor.
  • S2. Partner Schedule Set Up – used to define or maintain the schedule / availability for a partner (for example, working days, hours, and other scheduling rules).

The decision steps (D1, D2, etc.) guide the user to the correct path based on the type of setup needed.

4. What is the difference between a Work Code (S3) and a Partner Schedule (S2)?

A partner schedule (S2) answers “when and how is this partner available?” (calendar and availability).

A work code (S3. Create A Work Code) answers “what type of work is being performed?” (for example, Start-Up, Inspection, Warranty Service, Rental Return Check).

Partner schedules support planning and dispatching; work codes support how work is categorized, quoted, and billed.

5. How are warranty-specific GL accounts handled (S4)?

S4. Warranty Specific GL is used to define or adjust the General Ledger accounts associated with warranty work. This helps PPS:

  • Keep warranty revenue and costs separate from normal billable work.
  • Report accurately on warranty exposure and performance.
  • Reduce manual corrections in the financial system.

Having warranty GL correctly configured up front avoids confusion and rework later.

6. Why do we need SRO templates (S5)?

S5. Create an SRO Template is used to build reusable service order templates. These templates:

  • Pre-load common lines and defaults for specific job types (for example, standard start-up packages).
  • Reduce keying time and errors for coordinators and dispatchers.
  • Ensure key fields (work codes, warranty codes, rates) are applied consistently.

Templates are especially useful for high-volume or highly standardized service offerings.

7. What are Warranty Codes (S6) used for?

S6. Create A Warranty Code is used to define or maintain warranty codes that:

  • Identify the warranty status of work being performed.
  • Drive special billing and GL rules where needed.
  • Support reporting on warranty activity and cost.

Clear warranty codes make it easier to separate warranty work from normal billable service.

8. Why are Rates (S7) important in ISM Setup?

S7. Set Up a Rate defines the rates that will be used in service work:

  • Labor rates by technician, skill level, or customer type.
  • Special project or contract-specific rates.
  • Other rate structures required by PPS.

These rates are referenced by work codes, SRO templates, and contracts so that billing calculations are correct and consistent across PPS Service.

9. How do Contract Templates (S8 and S9) fit into the process?

The SOP includes two paths for contract templates:

  • S8. Create a Contract Rental Template – for rental-style contracts (for example, generator rentals).
  • S9. Create a Service Contract Template – for service or maintenance contracts (for example, PMA-type agreements).

These templates help standardize terms, billing patterns, and included services for contract-based work.

10. What is S998 – Restart Selection, and when is it used?

S998. Restart Selection is a convenience step that allows the user to:

  • Return to the ISM Setup decision hub without leaving the SOP.
  • Choose another setup path (for example, after setting up a partner, move on to rates or contract templates).

This reinforces the idea that ISM Setup is a menu-style hub for configuration work, letting you perform multiple related setup tasks in one session.
