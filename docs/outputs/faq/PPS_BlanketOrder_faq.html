PPS – Blanket Order Creation & Processing – Frequently Asked Questions

This FAQ supports the PPS Blanket Order Creation and Processing SOP (M1 – Blanket Order Creation and Processing – PPS Sales). It explains the main decisions and steps from the contractor’s initial order request through the final handoff to the Start-Up SOP.

Questions & Answers

1. What is the purpose of the PPS Blanket Order SOP?

The PPS Blanket Order SOP defines the end-to-end process for handling blanket orders for generator projects in PPS Sales. It covers:

  • The contractor’s initial request for a generator (with or without a transfer switch).
  • Creation of the blanket order in the system.
  • Handling non-stock / modified-build generators and serialization.
  • Creating and managing releases (01, 02, etc.).
  • Shipping feedback and AR for each release.
  • Handoff to the Start-Up SOP once the generator is ready for start-up.
2. Where does this SOP start and end?

The process starts at S1. Contractor Places an Order for a Generator, including the decision about whether the generator and transfer switch will be shipped together or separately.

It ends after the planned releases have been shipped and invoiced (for example 01 and 02 suffixes), and the process hands off to S17. Start-Up SOP when the customer calls for start-up (S16).

3. Why is the decision about separate shipment (D1) important?

The decision “Do they want the generator and transfer switch shipped separate?” (D1) drives how the blanket order and releases are structured:

  • If separate shipment is requested, additional releases may be created (for example, via S7. Create 3 Releases) so that the generator, transfer switch, and related work can move on different suffixes.
  • If combined shipment is acceptable, fewer releases may be used (for example, via S18. Create 2 Releases).

Getting this decision right up front reduces confusion around pick tickets and invoices later in the process.

4. How are modified-build generators handled (D3, S3, S4)?

When the answer to D3 – Is the generator a modified build ordered from vendor? is Yes:

  • S3. Create Order as Needed Non-Stock is used to set up the needed non-stock order for the generator.
  • S4. Make a Product Serialized ensures the generator is treated as a serialized item, supporting warranty, service history, and start-up tracking.

This ensures that modified-build units are properly tracked from vendor order through installation and start-up.

5. What do S5 (Add Items) and S6 (Add Lump Sum) do on the blanket order?

On the blanket order:

  • S5. Add Items (Including Start-Up) is used to add item lines such as the generator, transfer switch, start-up SKU, and any other related products.
  • S6. Add Lump Sum is used to add any lump-sum lines (for example, agreed package prices or additional charges that are not tied to a single item).

Together, these steps ensure that the blanket order accurately reflects what was sold to the customer, both in quantities and in dollars.

6. How do S7 and S18 (Create Releases) fit into the process?

The steps S7. Create 3 Releases and S18. Create 2 Releases control how the blanket order is broken into releases:

  • S7. Create 3 Releases can be used when multiple distinct releases are needed (for example, to separate transfer switch, generator, and other components).
  • S18. Create 2 Releases can be used when fewer releases are sufficient (for example, one for initial materials and one for the generator).

These releases drive which suffixes (such as 01 and 02) show up on pick tickets, shipping feedback, and AR documents.

7. What is the role of S8 – Run OEEPC?

S8. Run OEEPC processes the blanket order through the order engine before the first release moves forward. Depending on configuration, it can:

  • Validate pricing and any engineered configuration rules.
  • Update order lines and release structures.
  • Help ensure that subsequent pick tickets and shipments run cleanly.

Running OEEPC at the right time reduces rework and exceptions during release processing.

8. How does the first release (01 suffix) flow work?

The first release typically uses the 01 suffix and follows this flow:

  • S9. Customer Calls For First Release – the customer requests the first part of the order.
  • S8. Run OEEPC (if not already run) – to ensure the release is valid.
  • S10. Print Pick Ticket on 01 Suffix – the warehouse picks and stages material.
  • S11. Shipping Feedback – actual shipped quantities are recorded.
  • S12. “01 Suffix” Goes Through AR Process – the first release is invoiced.

This creates a clean chain from request to shipment to invoice for the first portion of the blanket order.

9. What happens in the second release and how does it link to Start-Up?

After the first release is complete:

  • S13. Customer Calls For Second Release (Generator) – the generator is now ready to be delivered.
  • A check is made on whether the expected number of releases were created (for example via D2 – “Were 3 Releases Created?”).
  • S14. Print Pick Ticket on 02 Suffix – the generator is picked.
  • S15. Shipping Feedback – the generator shipment is recorded.
  • S16. Customer Calls For Start-Up – once the generator is installed or ready.
  • S17. Start-Up SOP – the process hands off to the Start-Up SOP for the field start-up work.

This creates a clean bridge between sales / blanket order processing and PPS Service’s start-up activities.

10. How does the Blanket Order SOP fit into the broader PPS / Scott Electric process?

The Blanket Order SOP is one part of the overall generator project lifecycle:

  • Upstream: PPS Sales works with the contractor to define and price a solution (generator, transfer switch, start-up, etc.).
  • This SOP: The blanket order is created, configured, released, shipped, and invoiced in a controlled way.
  • Downstream: Once the generator has been shipped and the customer is ready, the Start-Up SOP handles field start-up, and future PMA or service work may follow.

A well-managed blanket order process keeps shipments, invoices, and start-up timing aligned, reducing surprises for both PPS and the customer.
